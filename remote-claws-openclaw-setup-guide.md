# Remote Claws + OpenClaw Setup Guide

Connect a **Remote Claws** MCP server (running on a Windows machine) to an OpenClaw agent. Once set up, the agent can control the remote desktop, run shell commands, automate a Chromium browser, and read/write files — exactly as a human sitting at that keyboard would.

---

## What You Need

| Item | Notes |
|------|-------|
| Windows machine | Must be reachable from your OpenClaw server (Tailscale strongly recommended) |
| Remote Claws | Running on that Windows machine, port 3030 by default |
| Bearer token | Generated by `remote-claws-setup` on the Windows machine |
| OpenClaw server | Linux/Mac |

---

## Step 1 — Install and Start Remote Claws on Windows

```powershell
git clone https://github.com/wentbackward/remote-claws.git
cd remote-claws
python -m venv .venv
.venv\Scripts\activate
pip install -e .
playwright install chromium
```

Generate your bearer token (copy it — it is only shown once):

```powershell
remote-claws-setup
```

Start the server:

```powershell
remote-claws
```

The server listens on `0.0.0.0:3030` by default.

---

## Step 2 — Configure allowed_hosts on the Windows Machine

Remote Claws validates the HTTP `Host` header. When an agent connects from a remote IP (e.g. over Tailscale), the Host header will contain that IP rather than `localhost`. You need to tell Remote Claws to accept it.

Create or edit `remote-claws.json` in the Remote Claws directory:

```json
{
  "allowed_hosts": "*"
}
```

This disables host checking. Safe to use when running behind Tailscale or a VPN. If you prefer to restrict it, list specific IPs:

```json
{
  "allowed_hosts": "localhost,127.0.0.1,100.106.2.100"
}
```

Replace `100.106.2.100` with your OpenClaw server's Tailscale IP.

Restart Remote Claws after saving the file.

### Verify it's reachable from your OpenClaw server

```bash
curl -s -H "Authorization: Bearer YOUR_TOKEN" \
  http://<WINDOWS_TAILSCALE_IP>:3030/sse --max-time 5
# Expected: event: endpoint / data: /messages/?session_id=...
```

If you get `401` — wrong token. If you get `421` or `Invalid Host header` — `allowed_hosts` not set or Remote Claws not restarted.

---

## Step 3 — Register the MCP Server in OpenClaw Config

Edit `~/.openclaw/openclaw.json`. Add the `mcp` block — merge carefully if the file already has content.

```json
{
  "mcp": {
    "servers": {
      "remote-claws": {
        "url": "http://<WINDOWS_TAILSCALE_IP>:3030/sse",
        "headers": {
          "Authorization": "Bearer YOUR_TOKEN_HERE"
        }
      }
    }
  },
  "commands": {
    "mcp": true
  }
}
```

Replace `<WINDOWS_TAILSCALE_IP>` with your Windows machine's Tailscale IP and `YOUR_TOKEN_HERE` with the token from Step 1.

### Restart the gateway

```bash
openclaw gateway restart
```

The agent will have access to all 39 Remote Claws tools on the next turn.

## Step 4 — Install the Skill

The skill file tells the agent what tools are available and when to use them. Without it, a weaker model may not know Remote Claws tools exist.

### Option A — Install from ClawHub (easiest)

```bash
openclaw skills install remote-claws
```

Or from chat:

```
/skills install remote-claws
```

### Option B — Create it manually

```bash
mkdir -p ~/workspace/skills/remote-claws
```

Save this as `~/workspace/skills/remote-claws/SKILL.md`:

```markdown
---
name: remote-claws
description: "Full remote desktop control of a Windows machine via Remote Claws MCP. Use when asked to: take a screenshot of the remote desktop; click, type, or drag with the mouse/keyboard on the remote machine; run commands or scripts on Windows; automate a Chromium browser on the remote machine; read or write files on the remote machine."

# Remote Claws — Remote Desktop Control

Controls a Windows machine over MCP/SSE. All 39 tools are provided by the
remote-claws MCP server registered in openclaw.json. No local proxy required.

## Verify connection

Before using tools, confirm the MCP server is reachable:

  curl -s -H "Authorization: Bearer <token>" http://<WINDOWS_IP>:3030/sse --max-time 5

Should return: event: endpoint / data: /messages/?session_id=...

## Tool Groups

### Desktop (mouse, keyboard, screenshots)
- desktop_screenshot        — capture full screen or region [x, y, width, height]
- desktop_mouse_click       — left/right/middle click at x, y
- desktop_mouse_move        — move cursor to x, y
- desktop_mouse_drag        — drag from start_x,start_y to end_x,end_y
- desktop_type_text         — type ASCII text at current focus (ASCII only)
- desktop_press_key         — press key or combo: "enter", "ctrl+c", "alt+f4"
- desktop_scroll            — scroll at x,y; direction "up" or "down"
- desktop_find_window       — find windows by title or class_name substring
- desktop_focus_window      — bring window to foreground by title substring
- desktop_list_elements     — list UI controls (buttons, fields) inside a window
- desktop_click_element     — click a named UI element (more reliable than coords)
- desktop_get_element_text  — read the value of a named UI element

### Browser (Chromium via Playwright — CSS selectors)
- browser_navigate          — go to a URL; returns title, final URL, status
- browser_click             — click element by CSS selector
- browser_fill              — set input value (handles Unicode, triggers change events)
- browser_type              — type keystroke-by-keystroke (appends, does not clear)
- browser_press_key         — key press e.g. "Enter", "Control+a"
- browser_get_text          — extract visible text from element (default: body)
- browser_get_html          — get HTML markup of element
- browser_eval_js           — run JavaScript in page context
- browser_screenshot        — screenshot page or element
- browser_wait_for          — wait for element: visible/hidden/attached/detached
- browser_select_option     — select a <select> dropdown option by value or label
- browser_go_back / browser_go_forward
- browser_tabs_list / browser_tab_new / browser_tab_close

### Exec (run commands on Windows, async)
- exec_run                  — start command; returns process_id immediately
- exec_get_output           — read stdout/stderr; set wait=true to block
- exec_send_input           — send a line to stdin of a running process
- exec_kill                 — terminate a process
- exec_list                 — list all tracked processes

### Files (base64 encoded)
- file_write                — write base64 content to a path
- file_read                 — read file as base64 (use offset/limit for large files)
- file_list                 — list directory; supports glob patterns, recursive
- file_delete               — delete file or empty directory
- file_move                 — move or rename file/directory
- file_info                 — get size, created, modified timestamps

## Important notes
- Screenshots are JPEG max 1280x960. Coordinates are absolute pixels.
  Take a fresh screenshot before clicking if the window has moved.
- desktop_type_text is ASCII only. For Unicode text, use browser_fill,
  or paste via clipboard: exec_run powershell Set-Clipboard, then desktop_press_key ctrl+v
- File content is base64 encoded. Decode after reading.
```

## Testing

Once configured, ask your agent:

> "Take a screenshot of my Windows desktop."

Then:

> "Open Notepad and type 'Hello World'."

If the screenshot comes back and Notepad opens with text in it, you're fully operational.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `421 Misdirected Request` or `Invalid Host header` | `allowed_hosts` not set | Add `"allowed_hosts": "*"` to `remote-claws.json`, restart Remote Claws |
| `401 Unauthorized` | Wrong or missing bearer token | Check the token in `openclaw.json` matches what `remote-claws-setup` printed |
| `Connection refused` or timeout | Remote Claws not running | Start it: `remote-claws` in the venv |
| Tools not available to agent | MCP config not loaded | Restart OpenClaw gateway |
| `desktop_type_text` drops characters | Non-ASCII input | Use `browser_fill` or the clipboard method |
| Screenshot coordinates wrong | Window moved since last screenshot | Take a fresh `desktop_screenshot` before clicking |

## Security Notes

- Use **Tailscale** or a VPN between your OpenClaw server and the Windows machine. Do not expose port 3030 to the public internet.
- `allowed_hosts: "*"` is safe on a private Tailscale network. On any public-facing machine, restrict it to specific IPs.
- The bearer token in `openclaw.json` has full desktop access. Keep the file permissions tight and do not commit it to a public repo.
- Remote Claws has no further auth beyond the token. Treat it like SSH access to that machine.
- Use `permissions.json` on the Windows side to restrict which tool groups the agent can use (browser, desktop, exec, files).
