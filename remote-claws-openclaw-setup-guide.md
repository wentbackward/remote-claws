# Remote Claws + OpenClaw Setup Guide

Connect a **Remote Claws** MCP server (running on a Windows machine) to an OpenClaw agent. For full documentation on Remote Claws itself, see the [README](README.md).

---

## What You Need

| Item | Notes |
|------|-------|
| Windows machine | Must be reachable from your OpenClaw server (Tailscale recommended) |
| Remote Claws | Installed and running on that machine — see [README](README.md#quick-start) |
| Bearer token | Generated by `remote-claws-setup` (shown once, copy it) |
| OpenClaw server | Linux/Mac/Windows |

---

## Step 1 — Install and Start Remote Claws on Windows

```powershell
git clone https://github.com/wentbackward/remote-claws.git
cd remote-claws
.\start.ps1
```

The `start.ps1` script handles first-run setup (venv, dependencies, Playwright, auth token) and starts the server. Copy the bearer token when it appears — it is only shown once.

The server runs in the **foreground** — keep the terminal open. You'll see tool calls logged as agents use them.

If you prefer to set up manually, see the [README](README.md#quick-start).

---

## Step 2 — Configure Remote Claws for Remote Access

By default the server only accepts connections where the Host header is `localhost`. When OpenClaw connects over Tailscale, you need to allow its IP.

Create or edit `remote-claws.json` in the remote-claws directory:

```json
{
  "allowed_hosts": "*"
}
```

This disables host checking — safe behind Tailscale or a VPN. To restrict to specific IPs:

```json
{
  "allowed_hosts": "localhost,127.0.0.1,100.106.2.100",
  "allowed_ips": "100.106.2.100"
}
```

Replace `100.106.2.100` with your OpenClaw server's Tailscale IP. See the [README](README.md#configuration) for all config options.

Restart Remote Claws after saving the file.

### Verify connectivity from your OpenClaw server

```bash
curl -s -H "Authorization: Bearer YOUR_TOKEN" \
  http://<WINDOWS_TAILSCALE_IP>:8080/sse --max-time 5
# Expected: event: endpoint / data: /messages/?session_id=...
```

## Step 3 — Register the MCP Server in OpenClaw

Edit `~/.openclaw/openclaw.json`. Merge carefully if the file already has content.

```json
{
  "mcp": {
    "servers": {
      "remote-claws": {
        "url": "http://<WINDOWS_TAILSCALE_IP>:8080/sse",
        "headers": {
          "Authorization": "Bearer YOUR_TOKEN_HERE"
        }
      }
    }
  },
  "commands": {
    "mcp": true
  }
}
```

Restart the gateway:

```bash
openclaw gateway restart
```

The agent will have access to all 39 Remote Claws tools on the next turn.

---

## Step 4 — Install the Skill

The skill file tells the agent what tools are available and when to use them. Without it, weaker models may not know Remote Claws tools exist.

### Option A — Install from ClawHub

```bash
openclaw skills install remote-claws
```

Or from chat:

```
/skills install remote-claws
```

### Option B — Create it manually

Copy `openclaw/SKILL.md` from this repository into your skills directory:

```bash
mkdir -p ~/workspace/skills/remote-claws
cp openclaw/SKILL.md ~/workspace/skills/remote-claws/SKILL.md
```

## Testing

Ask your agent:

> "Take a screenshot of my Windows desktop."

Then:

> "Open Notepad and type 'Hello, World!'"

If the screenshot comes back and Notepad opens with text in it, you're fully operational.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `421 Misdirected Request` | `allowed_hosts` not set | Add `"allowed_hosts": "*"` to `remote-claws.json`, restart |
| `401 Unauthorized` | Wrong or missing bearer token | Check the token in `openclaw.json` matches what `remote-claws-setup` printed |
| `403 Forbidden` | IP not in allowlist | Add OpenClaw server's IP to `allowed_ips`, or remove the setting to disable IP filtering |
| Connection refused / timeout | Remote Claws not running | Start it: `.\start.ps1` (keep the terminal open) |
| Tools not available to agent | MCP config not loaded | Restart OpenClaw gateway |

For additional troubleshooting, see the [README](README.md#troubleshooting-421-errors).
